# BookPath 웹 기능 정리 & 데이터 활용 전략

---

## 5가지 핵심 기능

### 🎯 기능 1: KDC 마인드맵 탐색
**목표**: 학생이 큰 분야 → 작은 분야로 드릴다운하며 책 탐색

| 항목 | 설명 |
|------|------|
| **필요 데이터** | `kdc_tree.json` |
| **데이터 가공** | 이미 완성 (000~900 계층) |
| **웹 구현** | D3.js 마인드맵 렌더링 |
| **사용자 경험** | 자연과학 → 물리학 → 고전역학 (계층 확장) |
| **클릭 결과** | 기능 2로 이동 (책 카드 표시) |

---

### 📚 기능 2: 책 카드 표시
**목표**: 선택된 분야의 도서 정보 + 학과 + 진로 한눈에 보기

| 항목 | 설명 | 데이터 출처 |
|------|------|-----------|
| **제목, 저자, 소개** | 원본 그대로 | `books_tagged.json` |
| **KDC 분야명** | "자연과학", "물리학" | `career_map[kdc].kdc_name` |
| **AI 태그** (학생 친화적 키워드) | ["양자역학", "입자", "우주"] | `books_tagged.json.student_tags` |
| **관련 학과 (상위 3개)** | 물리학과, 천문학과, 전자공학 | `career_map[kdc].departments[0:3].name` |
| **학과의 주요 교과목** | 고전역학, 양자역학, 전자기학 | `career_map[kdc].departments[].subjects` |
| **관련 진로** | 물리학연구원, 반도체기술자 | `career_map[kdc].departments[].careers` |
| **진로 정보 링크** | CareerNet 바로가기 | (수동 추가 필요) |

**카드 레이아웃**:
```
┌───────────────────────────┐
│ 책 제목                    │
│ 저자 | 출판사             │
├───────────────────────────┤
│ [#태그1] [#태그2] [#태그3]│
├───────────────────────────┤
│ 📍 분야: 자연과학(400)    │
├───────────────────────────┤
│ 🎓 관련 학과:            │
│   • 물리학과              │
│     - 고전역학, 양자역학  │
│     - 진로: 물리학연구원  │
│   • 천문학과              │
│   • 전자공학과            │
├───────────────────────────┤
│ ➜ 다른 시각으로 보기 [버튼]│
└───────────────────────────┘
```

### 🔄 기능 3: "다른 시각으로 보기" (옆 서가 이동)
**목표**: 현재 분야와 관련되지만 새로운 관점의 책 발견

| 항목 | 설명 | 로직 |
|------|------|------|
| **연결 선택** | 3개 중 랜덤 1개 | `side_shelf_map.connections[kdc]` |
| **제시 문구** | "과학이 만드는 기술" | `target.label` |
| **설명** | "기초과학이 공학 기술로..." | `target.reason` |
| **추천 책** | 3권 랜덤 | `books.filter(b => b.kdc==target_kdc)` + 태그 매칭 |

**예시 흐름**:
```
사용자가 물리학(420) 도서 보고 있음
→ "다른 시각으로 보기" 클릭
→ 3개 옵션 중 1개: "과학이 만드는 기술" (500. 기술과학)
→ 기술과학 분야의 책 3권 표시
  (예: 반도체, 나노기술, 인공지능)
```

**연결 관계 (총 30개)**:
- 400(자연과학) → 500(기술), 300(사회과학), 800(문학)
- 500(기술) → 400(자연과학), 600(예술-UX디자인), 100(철학)
- 등등...

### 📊 기능 4: 개인 독서 분석
**목표**: 학생의 읽은 책 기록 → 분야 성향 시각화

| 항목 | 설명 | 구현 |
|------|------|------|
| **책 등록** | "읽음" / "관심" 체크 | localStorage |
| **분야 비율 그래프** | 도넛 차트 (400:30%, 800:20%, ...) | Chart.js + KDC 그룹핑 |
| **태그 워드클라우드** | 자주 본 키워드 큰 글씨 | Word Cloud 라이브러리 |
| **탐색 현황** | "과학 많음, 철학 적음" | KDC별 읽은 책 수 |
| **추천** | "철학, AI윤리를 함께 보면..." | 미탐색 KDC 추천 |

**예시 분석**:
```
자연과학(400): 8권 (40%)
기술과학(500): 6권 (30%)
문학(800):     4권 (20%)
사회과학(300): 2권 (10%)

🔍 분석:
→ 이과 관심이 강합니다
→ 철학(100), 예술(600) 미탐색
→ AI 윤리, 과학철학 추천
```

### 🎯 기능 5: 진로 연결 정보
**목표**: 책 분야 → 대학 학과 → 구체적 직업 매핑

| 항목 | 설명 | 데이터 |
|------|------|--------|
| **학과명** | 물리학과, 천문학과, 전자공학 | `career_map[kdc].departments[].name` |
| **배우는 과목** | 고전역학, 양자역학, 전자기학, ... | `career_map[kdc].departments[].subjects` |
| **진로** | 물리학연구원, 반도체기술자, ... | `career_map[kdc].departments[].careers` |
| **외부 링크** | CareerNet 학과 상세정보 | (수동 추가) |

**활용 예시**:
```
책 "양자역학의 세계" 읽음
→ KDC 420 (물리학)
→ 관련 학과: 물리학과
  • 배우는 것: 고전역학, 양자역학, 광학, 통계역학
  • 진로: 물리학연구원, 반도체기술자
→ CareerNet 클릭 → 학과/직업 상세정보
```

## 데이터 가공 최종 정리

### 이미 완료된 가공
1. **raw_books.json** → 수집 완료
2. **kdc_tree.json** → 생성 완료 (KDC 계층)
3. **career_map.json** → 생성 완료 (학과 데이터)
4. **side_shelf_map.json** → 생성 완료 (30개 연결)

### 진행 중인 가공 (내일)
5. **books_tagged.json** → Gemini API로 완성 (KDC + student_tags + career_tags)

### 웹 구현에서 가공
6. **마인드맵 데이터** → `kdc_tree.json`을 D3.js로 렌더링
7. **책 필터링** → `books.filter(b => b.kdc.startsWith('4'))`
8. **태그 워드클라우드** → `books[읽은책].student_tags` 수집 + 빈도 계산
9. **연결 책 추천** → `side_shelf_map` + 책 필터링 + 태그 매칭
